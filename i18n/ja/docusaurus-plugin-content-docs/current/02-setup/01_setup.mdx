# セットアップ

## 1) 要件

- **SDKバージョン**: 0.3.6
- **最小SDK**: 26 (Android 8.0)
- **ターゲットSDK**: 36
- **対応ABI**: arm64-v8a, armeabi-v7a

---

### リソース

SDKにはビデオファイル（`.mp4`）とフェイスモデルファイル（`.npz`）が必要です。
これらのファイルを`assets`（`src/main/assets`）フォルダに配置し、`ResourceManager.init()`を呼び出してキャッシュコピーおよび初期化を行います。

```kotlin
import io.klleon.ondevice.ResourceManager

val resourceManager = ResourceManager()

ResourceManager.init(
    context = this,
    mp4FileName = "loop_sample.mp4",
    npzFileName = "loop_sample.npz",
    avatarId = "character_001",
    onSuccess = {
        // リソース準備完了 — play()の呼び出しが可能です
    },
    onError = { error ->
        Log.e("Setup", "リソース準備失敗", error)
    }
)
```

:::info
`init()`完了後、内部パスが自動的に設定されます。
`play()`を呼び出す前に、必ず`init()`の`onSuccess`コールバックが呼び出される必要があります。
:::

## 2) Android設定

```kotlin
android {
    compileSdk = 36

    defaultConfig {
        minSdk = 26
        targetSdk = 36
    }

    packaging {
        jniLibs {
            pickFirsts += "**/libc++_shared.so"
        }
    }
}
```

## 3) Gradle依存関係

アプリレベルの`build.gradle.kts`に以下の依存関係を追加します：

```kotlin
dependencies {
    // SDK
    implementation("io.klleon:ondevice-sdk:0.3.6")

    // Coroutines（SDK必須）
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.9.0")
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-android:1.9.0")
}
```

:::info バンドルライブラリ
SDK AARには以下のライブラリが含まれています：
- OkHttp, Okio, Retrofit
- Jackson (Core, Module)
- TensorFlow Lite
- OpenCV
- PyTorch Mobile
- Oboe
- ONNX Runtime
- Agora RTC

アプリでOkHttpまたはOkioを使用するライブラリ（例：Coil）がある場合、バージョン競合が発生する可能性があります。その場合、`RuntimeClasspath`から該当グループをexcludeしてください：
```kotlin
configurations.matching { it.name.contains("RuntimeClasspath", ignoreCase = true) }.configureEach {
    exclude(group = "com.squareup.okhttp3")
    exclude(group = "com.squareup.okio")
}
```
:::

## 4) ProGuard / R8

SDK AARに必要なProGuard/R8ルールが含まれているため、**追加の設定は不要です。**

`UnsatisfiedLinkError`が発生した場合は、上記Android設定の`jniLibs.pickFirsts`設定を確認してください。
