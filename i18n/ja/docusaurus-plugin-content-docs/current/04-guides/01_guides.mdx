### 1. 権限管理
- SDKの初期化前に必ず権限を確認してください
- ユーザーフレンドリーなメッセージで権限の拒否を処理してください
- 必要な権限に対する明確な理由を提示してください
- さまざまなAndroidバージョンで権限フローをテストしてください

```kotlin
private fun showPermissionRationale() {
    Toast.makeText(
        this,
        "このアプリが正常に動作するにはオーディオ権限が必要です。",
        Toast.LENGTH_LONG
    ).show()
}
```

### 2. リソース管理

`ResourceManager.init()`でリソースを初期化し、`onSuccess`コールバック後にのみ`play()`を呼び出してください。

#### 呼び出し順序

```
アバター選択   → ResourceManager.init(avatarId) → onSuccess → sdk.play() → ... → sdk.finish()
同じアバター   → ResourceManager.init(同一avatarId) → バッファ再利用 → 即座にonSuccess → sdk.play()
アバター変更   → ResourceManager.init(新avatarId) → 自動で以前のバッファ解放 + 新規割り当て → onSuccess → sdk.play()
アプリ終了     → ResourceManager.deinit()（Application.onDestroyまたはActivity.onDestroyで呼び出し）
```

#### 安全なクリーンアップパターン

SDKを使用するActivityでは`onDestroy()`で`finish()`を呼び出してください：

```kotlin
// ChatActivity（SDK play画面）
override fun onDestroy() {
    super.onDestroy()
    // finish()はsuspend関数のためコルーチンで実行
    CoroutineScope(Dispatchers.Default).launch {
        klleonOndeviceSdk.finish()
    }
}
```

アバター選択画面のように`init()`のみ呼び出し、`play()`は別のActivityで呼び出す場合、
SDKが再生中でなければ`deinit()`でメモリを解放してください：

```kotlin
// CharacterActivity（アバター選択画面）
override fun onDestroy() {
    super.onDestroy()
    if (!KlleonOndeviceSdk.playing) {
        ResourceManager.deinit()
    }
}
```

#### バックグラウンドでのメモリ解放

アプリがバックグラウンドに遷移すると、約1GBのYUVバッファが不必要に占有されます。`Application`に`ComponentCallbacks2`を登録して自動解放してください：

```kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        registerComponentCallbacks(object : ComponentCallbacks2 {
            override fun onTrimMemory(level: Int) {
                if (level >= ComponentCallbacks2.TRIM_MEMORY_UI_HIDDEN) {
                    ResourceManager.deinit()
                }
            }
            override fun onConfigurationChanged(newConfig: Configuration) {}
            override fun onLowMemory() { ResourceManager.deinit() }
        })
    }
}
```

### 3. 状態の監視
- ViewModelを使用してSDKの状態変更を監視してください
- コルーチンスコープで状態の更新を収集してください
- try-catchブロックでJSON応答を安全にパースしてください
- UI変更のためにメインスレッドで状態の更新を処理してください

### 4. エラー処理
- SDKメソッドの呼び出しをtry-catchブロックで囲んでください
- デバッグのためにエラーを適切にログ出力してください
- 失敗した操作に対してユーザーフィードバックを提供してください
- SDKメソッドの呼び出し前に入力を検証してください

```kotlin
fun safeOperation() {
    try {
        klleonOndeviceSdk.sendMessage(message)
    } catch (e: Exception) {
        Log.e("SDK", "操作失敗", e)
        showUserError("操作に失敗しました。もう一度お試しください。")
    }
}
```

### 5. UI統合
- Composeで`AndroidView`を使用して`KlleonSdkView`を統合してください
- SDKメソッドの呼び出し前にビューが適切に初期化されていることを確認してください
- ビューのライフサイクルを適切に処理してください
- さまざまな画面サイズと向きでUIをテストしてください

### 6. 音声認識の統合

ボタンタップ時に音声認識を開始し、無音検出時に自動終了されます。

```kotlin
// 音声認識の開始
val success = klleonOndeviceSdk.startVoiceRecognition()

// 結果はsdkState Flowで受信
klleonOndeviceSdk.sdkState.collect { state ->
    val json = JSONObject(state)
    when (json.optString("chat_type")) {
        "STT_RESULT" -> {
            // 音声認識の最終結果
            val text = json.getString("message")
        }
        "STT_PARTIAL" -> {
            // 中間結果（リアルタイム表示用）
            val partialText = json.getString("message")
        }
        "STT_STATUS" -> {
            // 音声認識の状態変化（準備中/再接続中/準備完了）
            val statusMessage = json.getString("message")
            showToast(statusMessage)
        }
    }
}
```

### 7. 発話の中断

アバターが発話中の場合、即座に中断できます。

```kotlin
klleonOndeviceSdk.sendStopSpeech()
```

### 8. Android TV対応

TVデバイスでは以下の事項を考慮してください：

- **画面回転**: `KlleonSdkView.setRenderRotation(270)` で270度回転をサポート
- **リモコンキーマッピング**: 回転された画面に合わせて方向キーをマッピング
- **フォーカス管理**: TVの自動フォーカスによるキーボードの自動表示を防止

```kotlin
// TV用の画面回転
klleonSdkView.setRenderRotation(270)
```

## トラブルシューティング

### よくある問題

#### 1. 権限の拒否
**問題**: `RECORD_AUDIO`権限が許可されていない

**解決策**:
- `AndroidManifest.xml`に権限が宣言されていることを確認
- 権限処理ロジックを確認
- さまざまなAndroidバージョンでテスト

#### 2. SDK状態が更新されない
**問題**: 状態の監視が動作しない

**解決策**:
- ViewModelで状態の監視が適切に設定されていることを確認
- コルーチンスコープがアクティブであることを確認
- 状態の収集が開始されていることを確認

#### 3. ビューが準備されていない
**問題**: `play()`の呼び出し前に`KlleonSdkView`が初期化されていない

**解決策**:
- `AndroidView`ファクトリでビューの初期化を確認
- ビューが準備された時点でのみボタンを有効化
- SDKメソッドの呼び出し前にnullチェックを追加

#### 4. USBマイクが認識されない
**問題**: USBマイクを接続したがSDKで使用されない

**解決策**:
- `hasUsbMicrophone()`で接続状態を確認
- MMAPポリシーはSDK内部で管理されます（デフォルト: 無効）。一部のUSBマイクはMMAP非互換です
- デバイスログを確認: `adb logcat | grep "Ondevice_Mic"`

#### 5. アバター発話中に音声認識ができない
**問題**: アバターが話している間、音声認識が開始されない

**解決策**:
- エコー防止のため、アバター発話中は音声認識がブロックされます
- `sendStopSpeech()`でアバターの発話をまず中断するか、発話完了後にお試しください

### デバッグのヒント

#### 状態のモニタリング
```kotlin
// すべての状態変更をモニタリング
klleonOndeviceSdk.sdkState.collect { state ->
    Log.d("SDKState", "状態更新: $state")

    try {
        val json = JSONObject(state)
        val message = json.getString("message")
        val chatType = json.optString("chat_type", "unknown")
        Log.d("SDKState", "パース結果 - chatType: $chatType, メッセージ: $message")
    } catch (e: Exception) {
        Log.w("SDKState", "状態JSONのパースに失敗", e)
    }
}
```

#### レコーダー状態のモニタリング
```kotlin
// レコーダーの状態変更をモニタリング
klleonOndeviceSdk.recorderState.collect { state ->
    when (state) {
        is OboeRecorderState.Recording -> {
            Log.d("Recorder", "録音中 - デバイス: ${state.deviceName}")
        }
        is OboeRecorderState.Error -> {
            Log.e("Recorder", "エラー: ${state.message}")
        }
        else -> Log.d("Recorder", "状態: $state")
    }
}
```
