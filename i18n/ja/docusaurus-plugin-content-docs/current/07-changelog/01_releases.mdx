# リリースノート

Klleon Ondevice SDKのバージョン別変更履歴です。

---

## v0.5.0 (Latest)

**リリース日**: 2026年4月

### Breaking Changes
- `pauseVideo()` / `resumeVideo()` / `isVideoPaused()` → `enterStandby()` / `exitStandby()` / `isStandby()`にリネーミング
- `registerKeywords()` / `clearKeywords()` / `keywordDetected` → `registerWakewords()` / `clearWakewords()` / `wakewordDetected`にリネーミング

### 新機能
- 待機画面（Standby）モード：パイプライン全体のflush + タイミング再初期化で早送り/ANR防止
- Wake-wordマッチング：STT認識結果から登録済みキーワードを検出 → コールバック伝達（standby中のみ有効）
- `setIdleTimeout()` / `disableIdleTimeout()`：TTS_END後の無活動時に自動待機画面進入
- `isResponding: StateFlow<Boolean>`：アバター応答状態追跡（PREPARING_RESPONSE ~ TTS_END）
- `STANDBY_ENTER`イベント：自動待機画面進入時にsdkStateへ発行

### バグ修正
- VideoDecoder `stop()` race condition（pauseToken/stopTokenの順序逆転）
- ASYNCモードconsumerがpauseToken未確認
- AudioPlayer `pause()` / `resume()` 追加（ビデオのみ停止しオーディオが再生継続する問題を修正）

## v0.4.0

**リリース日**: 2026年4月

### Breaking Changes
- `play()`、`finish()` → `suspend fun`に変更（コルーチン内での呼び出しが必要です）
- `KlleonOndeviceSdk.stopping` companionプロパティ削除 → `isStopping` インスタンスプロパティに置き換え

### 変更点
- `BaseElement.stop()`から`runBlocking`を削除：`withTimeout(2s)` + 強制キャンセルフォールバック
- `stopMutex`導入：`stop()`の同時進入を防止（ANR/デッドロック/二重解放防止）
- `awaitFinish()`追加：高速再進入（finish → play）時のリソース競合防止
- 未使用の`bridgeJob`コードを削除

## v0.3.6

**リリース日**: 2026年4月

### 変更点
- `ResourceManager.init()`に`avatarId`パラメータを追加
- YUVバッファライフサイクル分離：同一アバター再進入時にバッファ再利用（0ms）
- アバター切り替え時の3秒クールダウン（低スペックTV SoC OOM防御）
- `finish()`時にバッファ保持（`resetState`）+ アプリ終了時に完全解放（`deinit`）
- `onTrimMemory` / `onLowMemory`によるバックグラウンドメモリ自動解放
- `init()`内部IOコルーチン切り替え（Mainスレッドブロッキング除去）
