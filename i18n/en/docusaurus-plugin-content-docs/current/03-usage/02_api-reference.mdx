# API Reference

## Core Components

### KlleonOndeviceSdk

The main SDK class that provides all functionality.

```kotlin
val klleonOndeviceSdk = KlleonOndeviceSdk()
```

### KlleonSdkView

A custom Android View for rendering SDK content.

```kotlin
@Composable
fun SdkViewComponent() {
    var workerView by remember { mutableStateOf<KlleonSdkView?>(null) }

    AndroidView(
        factory = { context ->
            KlleonSdkView(context).also { view ->
                workerView = view
            }
        },
        modifier = Modifier.fillMaxSize()
    )
}
```

#### setRenderRotation()

Used when screen rotation is needed, such as on TVs.

```kotlin
klleonSdkView.setRenderRotation(270) // Supports 0, 90, 270 degrees
```

## SDK Methods

### play()

Starts AI avatar video playback.

```kotlin
klleonOndeviceSdk.play(
    context: Context,
    klleonSdkViewInstance: KlleonSdkView?,
    sdkKey: String,
    characterId: String,
    language: String,
    mmapEnabled: Boolean = false
)
```

**Parameters:**
| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `context` | Context | Yes | - | Current context |
| `klleonSdkViewInstance` | KlleonSdkView? | Yes | - | Rendering view |
| `sdkKey` | String | Yes | - | SDK authentication key |
| `characterId` | String | Yes | - | AI avatar character ID |
| `language` | String | Yes | - | Language code (ko, en, ja, etc.) |
| `mmapEnabled` | Boolean | No | false | Enable audio MMAP |

**Example:**
```kotlin
workerView?.let { view ->
    klleonOndeviceSdk.play(
        context = this,
        klleonSdkViewInstance = view,
        sdkKey = "YOUR_SDK_KEY",
        characterId = "character_001",
        language = "ko"
    )
}
```

### connect()

Performs WebSocket reconnection.

```kotlin
klleonOndeviceSdk.connect()
```

### finish()

Stops the SDK and releases all resources.

```kotlin
klleonOndeviceSdk.finish()
```

## Voice Recognition API

### startVoiceRecognition()

Starts voice recognition. Begins microphone recording and sends a START_VOICE signal via WebSocket. When silence is detected, recording automatically stops and an END_VOICE signal is sent.

```kotlin
fun startVoiceRecognition(
    config: SilenceDetectionConfig = SilenceDetectionConfig.default()
): Boolean
```

**Parameters:**
- `config`: Silence detection configuration (optional, default: `SilenceDetectionConfig.default()`)

**Returns:** Success status (Boolean)

**Example:**
```kotlin
// Start voice recognition with default settings
val success = klleonOndeviceSdk.startVoiceRecognition()

// Start with custom settings
val customConfig = SilenceDetectionConfig.default().copy(
    silenceTimeoutMs = 2000L  // 2-second silence detection timeout
)
val success = klleonOndeviceSdk.startVoiceRecognition(customConfig)
```

### stopVoiceRecognition()

Stops voice recognition.

```kotlin
klleonOndeviceSdk.stopVoiceRecognition()
```

### isVoiceRecognizing()

Checks whether voice recognition is in progress.

```kotlin
val isRecognizing: Boolean = klleonOndeviceSdk.isVoiceRecognizing()
```

## State Management

### sdkState

A SharedFlow that emits SDK state updates as JSON strings.

```kotlin
klleonOndeviceSdk.sdkState: SharedFlow<String>
```

**Usage:**
```kotlin
klleonOndeviceSdk.sdkState.collect { state ->
    val messageToAdd = try {
        val json = JSONObject(state)
        json.getString("message")
    } catch (t: Throwable) {
        state.toString()
    }

    // Handle state update
    updateUI(messageToAdd)
}
```

**State Format (JSON):**
```json
{
  "message": "",
  "chat_type": "ACTIVATE_VOICE",
  "time": "2025-09-03T05:42:08.394975295",
  "id": "99dd3ab3-53c4-4418-bee4-b1fbaea272aa"
}
```

**Key chat_type Values:**
| chat_type | Description |
|-----------|-------------|
| `WSS_OPEN` | WebSocket connection established |
| `WSS_FAILURE` | WebSocket connection failed |
| `WSS_CLOSING` | WebSocket connection closing |
| `WSS_CLOSED` | WebSocket connection closed |
| `WSS_RETRY_OVER` | Maximum retry attempts exceeded |
| `WSS_NOT_CONNECTED` | Not connected |
| `ACTIVATE_VOICE` | Voice activated |
| `RESPONSE` | Avatar response message |
| `STT_RESULT` | Voice recognition result |
| `STT_PARTIAL` | Voice recognition partial result |
| `STT_ERROR` | Voice recognition error |

### recorderState

A StateFlow for subscribing to recorder state changes in real time.

```kotlin
klleonOndeviceSdk.recorderState: StateFlow<OboeRecorderState>
```

**State Values:**
| State | Description |
|-------|-------------|
| `Idle` | Initial state |
| `Starting` | Starting up |
| `Recording(deviceId, deviceName, deviceType)` | Recording in progress |
| `Stopping` | Stopping |
| `Error(message)` | Error state |

### silenceDetectionResult

A SharedFlow that emits silence detection results.

```kotlin
klleonOndeviceSdk.silenceDetectionResult: SharedFlow<SilenceDetectionResult>?
```

**Result Types:**
| Type | Description |
|------|-------------|
| `SpeechTimeout` | Voice input wait timeout exceeded |
| `SilenceDetected` | Silence detected after speech (recording auto-stopped) |
| `MaxTimeReached` | Maximum recording time exceeded |
| `ManualStop` | Manual stop |
| `Error` | Error |

## SilenceDetectionConfig

Configures silence detection behavior during voice recognition.

```kotlin
import io.klleon.ondevice.audio.oboe.SilenceDetectionConfig
```

**Fields:**
| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `silenceThreshold` | Float | 0.02 | Level threshold for silence detection |
| `speechThreshold` | Float | 0.05 | Level threshold for speech detection |
| `speechTimeoutMs` | Long | 5000 | Voice input wait timeout (ms) |
| `silenceTimeoutMs` | Long | 2000 | Silence detection timeout after speech (ms) |
| `maxRecordingTimeMs` | Long | 60000 | Maximum recording time (ms, 0=unlimited) |
| `preBufferMs` | Int | 300 | Audio preserved before speech start (ms) |
| `preBufferDelayMs` | Long | 300 | Pre-buffer flush delay (ms) |

**Presets:**
| Preset | Characteristics |
|--------|-----------------|
| `SilenceDetectionConfig.default()` | Default settings |
| `SilenceDetectionConfig.fast()` | Fast response (speechTimeout 3s, silenceTimeout 1.5s) |
| `SilenceDetectionConfig.longSpeech()` | Long speech (speechTimeout 7s, silenceTimeout 3s, maxRecording 120s) |
| `SilenceDetectionConfig.noisyEnvironment()` | Noisy environment (higher thresholds: silence 0.05, speech 0.10) |

**Example:**
```kotlin
// Using presets
klleonOndeviceSdk.startVoiceRecognition(SilenceDetectionConfig.fast())

// Custom configuration
val config = SilenceDetectionConfig.default().copy(
    speechTimeoutMs = 3000L,
    maxRecordingTimeMs = 120_000L
)
klleonOndeviceSdk.startVoiceRecognition(config)
```

## Error Handling

All SDK methods should be wrapped in try-catch blocks:

```kotlin
try {
    klleonOndeviceSdk.sendMessage(message)
} catch (e: Exception) {
    Log.e("KlleonSDK", "Operation failed", e)
    // Handle error appropriately
}
```

## Lifecycle Management

```kotlin
class MainActivity : ComponentActivity() {
    private val klleonOndeviceSdk = KlleonOndeviceSdk()

    override fun onDestroy() {
        super.onDestroy()
        klleonOndeviceSdk.finish() // Release resources
    }
}
```
