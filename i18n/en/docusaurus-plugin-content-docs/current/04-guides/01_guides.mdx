### 1. Permission Management
- Always check permissions before initializing the SDK
- Handle permission denials with user-friendly messages
- Provide clear rationale for required permissions
- Test the permission flow across different Android versions

```kotlin
private fun showPermissionRationale() {
    Toast.makeText(
        this,
        "Audio permission is required for this app to function properly.",
        Toast.LENGTH_LONG
    ).show()
}
```

### 2. Resource Management

Initialize resources with `ResourceManager.init()` and only call `play()` after the `onSuccess` callback.

#### Call Order

```
Avatar selected  → ResourceManager.init(avatarId) → onSuccess → sdk.play() → ... → sdk.finish()
Same avatar      → ResourceManager.init(same avatarId) → Buffer reused → Immediate onSuccess → sdk.play()
Avatar changed   → ResourceManager.init(new avatarId) → Automatically release previous buffer + allocate new → onSuccess → sdk.play()
App exit         → ResourceManager.deinit() (call in Application.onDestroy or Activity.onDestroy)
```

#### Safe Cleanup Pattern

In the Activity that uses the SDK, call `finish()` in `onDestroy()`:

```kotlin
// ChatActivity (SDK play screen)
override fun onDestroy() {
    super.onDestroy()
    klleonOndeviceSdk.finish()
}
```

In cases where only `init()` is called and `play()` is called in a different Activity (such as an avatar selection screen), release memory with `deinit()` if the SDK is not playing:

```kotlin
// CharacterActivity (avatar selection screen)
override fun onDestroy() {
    super.onDestroy()
    if (!KlleonOndeviceSdk.playing) {
        ResourceManager.deinit()
    }
}
```

#### Background Memory Release

When the app moves to the background, the ~1GB YUV buffer is unnecessarily retained. Register `ComponentCallbacks2` in your `Application` to release it automatically:

```kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        registerComponentCallbacks(object : ComponentCallbacks2 {
            override fun onTrimMemory(level: Int) {
                if (level >= ComponentCallbacks2.TRIM_MEMORY_UI_HIDDEN) {
                    ResourceManager.deinit()
                }
            }
            override fun onConfigurationChanged(newConfig: Configuration) {}
            override fun onLowMemory() { ResourceManager.deinit() }
        })
    }
}
```

### 3. State Observation
- Use a ViewModel to observe SDK state changes
- Collect state updates in a coroutine scope
- Safely parse JSON responses with try-catch blocks
- Handle state updates on the main thread for UI changes

### 4. Error Handling
- Wrap SDK method calls in try-catch blocks
- Log errors appropriately for debugging
- Provide user feedback for failed operations
- Validate inputs before calling SDK methods

```kotlin
fun safeOperation() {
    try {
        klleonOndeviceSdk.sendMessage(message)
    } catch (e: Exception) {
        Log.e("SDK", "Operation failed", e)
        showUserError("The operation failed. Please try again.")
    }
}
```

### 5. UI Integration
- Use `AndroidView` in Compose to integrate `KlleonSdkView`
- Ensure the view is properly initialized before calling SDK methods
- Handle view lifecycle appropriately
- Test the UI across different screen sizes and orientations

### 6. Voice Recognition Integration

Tap a button to start voice recognition; it automatically stops when silence is detected.

```kotlin
// Start voice recognition
val success = klleonOndeviceSdk.startVoiceRecognition()

// Results are received via the sdkState Flow
klleonOndeviceSdk.sdkState.collect { state ->
    val json = JSONObject(state)
    when (json.optString("chat_type")) {
        "STT_RESULT" -> {
            // Final voice recognition result
            val text = json.getString("message")
        }
        "STT_PARTIAL" -> {
            // Interim result (for real-time display)
            val partialText = json.getString("message")
        }
        "STT_STATUS" -> {
            // Voice recognition status change (preparing/reconnecting/ready)
            val statusMessage = json.getString("message")
            showToast(statusMessage)
        }
    }
}
```

### 7. Speech Interruption

You can immediately stop the avatar's speech while it is speaking.

```kotlin
klleonOndeviceSdk.sendStopSpeech()
```

### 8. Android TV Support

Consider the following when targeting TV devices:

- **Screen rotation**: Support 270-degree rotation with `KlleonSdkView.setRenderRotation(270)`
- **Remote key mapping**: Map directional keys to match the rotated screen
- **Focus management**: Prevent automatic keyboard display caused by TV auto-focus

```kotlin
// Screen rotation for TV
klleonSdkView.setRenderRotation(270)
```

## Troubleshooting

### Common Issues

#### 1. Permission Denied
**Problem**: `RECORD_AUDIO` permission not granted

**Solution**:
- Verify the permission is declared in `AndroidManifest.xml`
- Check the permission handling logic
- Test across different Android versions

#### 2. SDK State Not Updating
**Problem**: State observation is not working

**Solution**:
- Verify that state observation is properly set up in the ViewModel
- Check that the coroutine scope is active
- Confirm that state collection has started

#### 3. View Not Ready
**Problem**: `KlleonSdkView` is not initialized before `play()` is called

**Solution**:
- Verify view initialization in the `AndroidView` factory
- Enable the button only when the view is ready
- Add null checks before calling SDK methods

#### 4. USB Microphone Not Recognized
**Problem**: USB microphone is connected but not used by the SDK

**Solution**:
- Check connection status with `hasUsbMicrophone()`
- MMAP policy is managed internally by the SDK (default: disabled). Some USB microphones are incompatible with MMAP
- Check device logs: `adb logcat | grep "Ondevice_Mic"`

#### 5. Voice Recognition Not Working During Avatar Speech
**Problem**: Voice recognition does not start while the avatar is speaking

**Solution**:
- Voice recognition is blocked during avatar speech to prevent echo
- Either stop the avatar's speech first with `sendStopSpeech()`, or try after the speech completes

### Debugging Tips

#### State Monitoring
```kotlin
// Monitor all state changes
klleonOndeviceSdk.sdkState.collect { state ->
    Log.d("SDKState", "State update: $state")

    try {
        val json = JSONObject(state)
        val message = json.getString("message")
        val chatType = json.optString("chat_type", "unknown")
        Log.d("SDKState", "Parsed - chatType: $chatType, message: $message")
    } catch (e: Exception) {
        Log.w("SDKState", "Failed to parse state JSON", e)
    }
}
```

#### Recorder State Monitoring
```kotlin
// Monitor recorder state changes
klleonOndeviceSdk.recorderState.collect { state ->
    when (state) {
        is OboeRecorderState.Recording -> {
            Log.d("Recorder", "Recording - device: ${state.deviceName}")
        }
        is OboeRecorderState.Error -> {
            Log.e("Recorder", "Error: ${state.message}")
        }
        else -> Log.d("Recorder", "State: $state")
    }
}
```
