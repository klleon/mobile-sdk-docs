### 1. Permission Management
- Always check permissions before SDK initialization
- Handle permission denials with user-friendly messages
- Provide clear rationale for required permissions
- Test permission flows on various Android versions

```kotlin
private fun showPermissionRationale() {
    Toast.makeText(
        this,
        "Audio permission is required for this app to function properly.",
        Toast.LENGTH_LONG
    ).show()
}
```

### 2. Resource Management

Initialize resources with `ResourceManager.init()` and only call `play()` after the `onSuccess` callback.

#### Call Order

```
Avatar selection → ResourceManager.init(avatarId) → onSuccess → sdk.play() → ... → sdk.finish()
Same avatar      → ResourceManager.init(same avatarId) → Buffer reuse → Instant onSuccess → sdk.play()
Avatar change    → ResourceManager.init(new avatarId) → Auto-release previous buffer + new allocation → onSuccess → sdk.play()
App exit         → ResourceManager.deinit() (call in Application.onDestroy or Activity.onDestroy)
```

#### Safe Cleanup Pattern

In Activities that use the SDK, call `finish()` in `onDestroy()`:

```kotlin
// ChatActivity (SDK play screen)
override fun onDestroy() {
    super.onDestroy()
    klleonOndeviceSdk.finish()
}
```

For screens like avatar selection where only `init()` is called and `play()` is called in another Activity,
release memory with `deinit()` if the SDK is not playing:

```kotlin
// CharacterActivity (avatar selection screen)
override fun onDestroy() {
    super.onDestroy()
    if (!KlleonOndeviceSdk.playing) {
        ResourceManager.deinit()
    }
}
```

#### Background Memory Release

When the app moves to the background, the ~1GB YUV buffer is unnecessarily occupied. Register `ComponentCallbacks2` in your `Application` for automatic release:

```kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        registerComponentCallbacks(object : ComponentCallbacks2 {
            override fun onTrimMemory(level: Int) {
                if (level >= ComponentCallbacks2.TRIM_MEMORY_UI_HIDDEN) {
                    ResourceManager.deinit()
                }
            }
            override fun onConfigurationChanged(newConfig: Configuration) {}
            override fun onLowMemory() { ResourceManager.deinit() }
        })
    }
}
```

### 3. State Observation
- Use ViewModel to observe SDK state changes
- Collect state updates in a coroutine scope
- Safely parse JSON responses with try-catch blocks
- Process state updates on the main thread for UI changes

### 4. Error Handling
- Wrap SDK method calls in try-catch blocks
- Log errors appropriately for debugging
- Provide user feedback for failed operations
- Validate inputs before calling SDK methods

```kotlin
fun safeOperation() {
    try {
        klleonOndeviceSdk.sendMessage(message)
    } catch (e: Exception) {
        Log.e("SDK", "Operation failed", e)
        showUserError("Operation failed. Please try again.")
    }
}
```

### 6. Voice Recognition Integration

Start voice recognition on button tap; it automatically stops when silence is detected.

```kotlin
// Start voice recognition
val success = klleonOndeviceSdk.startVoiceRecognition()

// Results are received via sdkState Flow
klleonOndeviceSdk.sdkState.collect { state ->
    val json = JSONObject(state)
    when (json.optString("chat_type")) {
        "STT_RESULT" -> {
            // Final speech recognition result
            val text = json.getString("message")
        }
        "STT_PARTIAL" -> {
            // Intermediate result (for real-time display)
            val partialText = json.getString("message")
        }
        "STT_STATUS" -> {
            // Speech recognition status change (preparing/reconnecting/ready)
            val statusMessage = json.getString("message")
            showToast(statusMessage)
        }
    }
}
```

Consider the following for TV devices:

- **Screen Rotation**: Support 270-degree rotation with `KlleonSdkView.setRenderRotation(270)`
- **Remote Key Mapping**: Map directional keys to match the rotated screen
- **Focus Management**: Prevent keyboard auto-display caused by TV auto-focus

```kotlin
// TV screen rotation
klleonSdkView.setRenderRotation(270)
```

## Troubleshooting

### Common Issues

#### 1. Permission Denied
**Problem**: `RECORD_AUDIO` permission not granted

**Solution**:
- Verify permission is declared in `AndroidManifest.xml`
- Check permission handling logic
- Test on various Android versions

#### 2. SDK State Not Updating
**Problem**: State observation not working

**Solution**:
- Verify state observation is properly set up in ViewModel
- Check that the coroutine scope is active
- Confirm state collection has started

#### 3. View Not Ready
**Problem**: `KlleonSdkView` not initialized before `play()` call

**Solution**:
- Verify view initialization in `AndroidView` factory
- Only enable buttons when the view is ready
- Add null checks before SDK method calls

#### 4. USB Microphone Not Recognized
**Problem**: USB microphone connected but not used by SDK

**Solution**:
- Check connection status with `hasUsbMicrophone()`
- MMAP policy is managed internally by the SDK (default: disabled). Some USB microphones are MMAP-incompatible
- Check device logs: `adb logcat | grep "Ondevice_Mic"`

#### 5. Voice Recognition Not Working During Avatar Speech
**Problem**: Voice recognition doesn't start while the avatar is speaking

**Solution**:
- Voice recognition is blocked during avatar speech for echo prevention
- Stop avatar speech first with `sendStopSpeech()`, or try after speech completes

### Debugging Tips

#### State Monitoring
```kotlin
// Monitor all state changes
klleonOndeviceSdk.sdkState.collect { state ->
    Log.d("SDKState", "State update: $state")

    try {
        val json = JSONObject(state)
        val message = json.getString("message")
        val chatType = json.optString("chat_type", "unknown")
        Log.d("SDKState", "Parsed - chatType: $chatType, message: $message")
    } catch (e: Exception) {
        Log.w("SDKState", "Failed to parse state JSON", e)
    }
}
```

#### Recorder State Monitoring
```kotlin
// Monitor recorder state changes
klleonOndeviceSdk.recorderState.collect { state ->
    when (state) {
        is OboeRecorderState.Recording -> {
            Log.d("Recorder", "Recording - device: ${state.deviceName}")
        }
        is OboeRecorderState.Error -> {
            Log.e("Recorder", "Error: ${state.message}")
        }
        else -> Log.d("Recorder", "State: $state")
    }
}
```
