### 1. Permission Management
- Always check permissions before SDK initialization
- Handle permission denials with user-friendly messages
- Provide clear rationale for required permissions
- Test permission flows on various Android versions

```kotlin
private fun showPermissionRationale() {
    Toast.makeText(
        this,
        "Audio permission is required for this app to work properly.",
        Toast.LENGTH_LONG
    ).show()
}
```

### 2. Resource Management

Initialize resources with `ResourceManager.init()` and only call `play()` after the `onSuccess` callback.

#### Call Order

```
Avatar selection -> ResourceManager.init(avatarId) -> onSuccess -> sdk.play() -> ... -> sdk.finish()
Same avatar      -> ResourceManager.init(same avatarId) -> Buffer reuse -> Immediate onSuccess -> sdk.play()
Avatar change    -> ResourceManager.init(new avatarId) -> Automatically release previous buffer + new allocation -> onSuccess -> sdk.play()
App exit         -> ResourceManager.deinit() (call in Application.onDestroy or Activity.onDestroy)
```

#### Safe Cleanup Pattern

In Activities that use the SDK, call `finish()` in `onDestroy()`:

```kotlin
// ChatActivity (SDK play screen)
override fun onDestroy() {
    super.onDestroy()
    // finish() is a suspend function, so run it in a coroutine
    CoroutineScope(Dispatchers.Default).launch {
        klleonOndeviceSdk.finish()
    }
}
```

In cases where only `init()` is called and `play()` is called in a different Activity (such as the avatar selection screen),
release memory with `deinit()` if the SDK is not playing:

```kotlin
// CharacterActivity (avatar selection screen)
override fun onDestroy() {
    super.onDestroy()
    if (!KlleonOndeviceSdk.playing) {
        ResourceManager.deinit()
    }
}
```

#### Background Memory Release

When the app transitions to the background, the ~1GB YUV buffer remains unnecessarily occupied. Register `ComponentCallbacks2` in your `Application` for automatic release:

```kotlin
class MyApplication : Application() {
    override fun onCreate() {
        super.onCreate()
        registerComponentCallbacks(object : ComponentCallbacks2 {
            override fun onTrimMemory(level: Int) {
                if (level >= ComponentCallbacks2.TRIM_MEMORY_UI_HIDDEN) {
                    ResourceManager.deinit()
                }
            }
            override fun onConfigurationChanged(newConfig: Configuration) {}
            override fun onLowMemory() { ResourceManager.deinit() }
        })
    }
}
```

### 4. Error Handling
- Wrap SDK method calls in try-catch blocks
- Log errors appropriately for debugging
- Provide user feedback for failed operations
- Validate inputs before calling SDK methods

```kotlin
fun safeOperation() {
    try {
        klleonOndeviceSdk.sendMessage(message)
    } catch (e: Exception) {
        Log.e("SDK", "Operation failed", e)
        showUserError("The operation failed. Please try again.")
    }
}
```

### 6. Voice Recognition Integration

Voice recognition starts on button tap and automatically ends when silence is detected.

```kotlin
// Start voice recognition
val success = klleonOndeviceSdk.startVoiceRecognition()

// Results are received via the sdkState Flow
klleonOndeviceSdk.sdkState.collect { state ->
    val json = JSONObject(state)
    when (json.optString("chat_type")) {
        "STT_RESULT" -> {
            // Voice recognition final result
            val text = json.getString("message")
        }
        "STT_PARTIAL" -> {
            // Interim result (for real-time display)
            val partialText = json.getString("message")
        }
        "STT_STATUS" -> {
            // Voice recognition state change (preparing/reconnecting/ready)
            val statusMessage = json.getString("message")
            showToast(statusMessage)
        }
    }
}
```

### 9. Standby Mode

Like a computer's screensaver, this feature transitions the avatar to standby after a period of inactivity and returns via wake-word.

#### Flow

```
play() -> setIdleTimeout() -> Conversation -> TTS_END -> 60 seconds of inactivity
  -> STANDBY_ENTER event -> Automatic enterStandby()
  -> Wake-word detected -> exitStandby() + language switch -> Resume conversation
```

#### Automatic Standby Setup

```kotlin
// Enable idle timeout after play()
klleonOndeviceSdk.play(context, view, sdkKey, avatarId, language)
klleonOndeviceSdk.setIdleTimeout(60_000L)  // Auto standby after 60 seconds of inactivity
```

If not called, the automatic standby feature remains disabled.

#### Exiting Standby with Wake-word

By combining with wakeword matching, standby can be exited via voice commands. Wakeword matching is automatically enabled upon `enterStandby()` and automatically disabled upon `exitStandby()`, so no separate on/off control is needed from the app.

```kotlin
// Register wakewords (for language switching) — registration alone does not activate matching
klleonOndeviceSdk.registerWakewords(mapOf(
    "ジャパニーズ" to "ja_jp",
    "イングリッシュ" to "en_us",
    "コリアン" to "ko_kr",
))

// Switch language + exit standby when wakeword is detected
// Since matching only works during standby, wakewordDetected is only emitted in standby state
klleonOndeviceSdk.wakewordDetected.collect { result ->
    klleonOndeviceSdk.sendChangeLanguage(result.tag)
    if (klleonOndeviceSdk.isStandby()) {
        klleonOndeviceSdk.exitStandby()  // Matching is also automatically disabled
    }
}
```

:::tip
Place `sendChangeLanguage()` outside the `isStandby()` guard. Under the current policy, wakewords are only matched during standby, but for defensive coding, it is better to place it outside the guard.
:::

:::caution
Wake-word detection only works when STT is active. Use `VoiceRecognitionMode.ALWAYS_ON` mode. In `DISCRETE` mode, you must exit standby via touch/button or other methods.
:::

#### Receiving STANDBY_ENTER Events

When automatic standby is entered, a `STANDBY_ENTER` event is emitted via `sdkState`. Apps can use this for UI updates (displaying overlays, etc.).

```kotlin
klleonOndeviceSdk.sdkState.collect { state ->
    if (state.contains("STANDBY_ENTER")) {
        // Display standby UI
    }
}
```

## Troubleshooting

### Common Issues

#### 1. Permission Denied
**Problem**: `RECORD_AUDIO` permission is not granted

**Solution**:
- Verify that permissions are declared in `AndroidManifest.xml`
- Check permission handling logic
- Test on various Android versions

#### 2. SDK State Not Updating
**Problem**: State observation is not working

**Solution**:
- Verify that state observation is properly set up in the ViewModel
- Ensure the coroutine scope is active
- Confirm that state collection has been started

#### 3. View Not Ready
**Problem**: `KlleonSdkView` is not initialized before calling `play()`

**Solution**:
- Verify view initialization in the `AndroidView` factory
- Only enable buttons when the view is ready
- Add null checks before calling SDK methods

#### 4. USB Microphone Not Recognized
**Problem**: USB microphone is connected but not used by the SDK

**Solution**:
- Check connection status with `hasUsbMicrophone()`
- MMAP policy is managed internally by the SDK (default: disabled). Some USB microphones are incompatible with MMAP
- Check device logs: `adb logcat | grep "Ondevice_Mic"`

#### 5. Voice Recognition Not Working During Avatar Speech
**Problem**: Voice recognition does not start while the avatar is speaking

**Solution**:
- Voice recognition is blocked during avatar speech to prevent echo
- Stop avatar speech first with `sendStopSpeech()`, or try after speech completes

### Debugging Tips

#### State Monitoring
```kotlin
// Monitor all state changes
klleonOndeviceSdk.sdkState.collect { state ->
    Log.d("SDKState", "State update: $state")

    try {
        val json = JSONObject(state)
        val message = json.getString("message")
        val chatType = json.optString("chat_type", "unknown")
        Log.d("SDKState", "Parsed - chatType: $chatType, message: $message")
    } catch (e: Exception) {
        Log.w("SDKState", "Failed to parse state JSON", e)
    }
}
```

#### Recorder State Monitoring
```kotlin
// Monitor recorder state changes
klleonOndeviceSdk.recorderState.collect { state ->
    when (state) {
        is OboeRecorderState.Recording -> {
            Log.d("Recorder", "Recording - device: ${state.deviceName}")
        }
        is OboeRecorderState.Error -> {
            Log.e("Recorder", "Error: ${state.message}")
        }
        else -> Log.d("Recorder", "State: $state")
    }
}
```
