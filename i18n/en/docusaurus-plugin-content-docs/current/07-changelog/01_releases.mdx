# Release Notes

Version history for Klleon Ondevice SDK.

---

## v0.5.0 (Latest)

**Release Date**: April 2026

### Breaking Changes
- `pauseVideo()` / `resumeVideo()` / `isVideoPaused()` renamed to `enterStandby()` / `exitStandby()` / `isStandby()`
- `registerKeywords()` / `clearKeywords()` / `keywordDetected` renamed to `registerWakewords()` / `clearWakewords()` / `wakewordDetected`

### New Features
- Standby mode: Full pipeline flush + timing reinitialization to prevent fast-forward/ANR
- Wake-word matching: Detect registered keywords from STT recognition results → callback delivery (active only during standby)
- `setIdleTimeout()` / `disableIdleTimeout()`: Automatic standby entry after inactivity following TTS_END
- `isResponding: StateFlow<Boolean>`: Avatar response state tracking (PREPARING_RESPONSE ~ TTS_END)
- `STANDBY_ENTER` event: Emitted to sdkState on automatic standby entry

### Bug Fixes
- VideoDecoder `stop()` race condition (pauseToken/stopToken order reversal)
- ASYNC mode consumer not checking pauseToken
- Added AudioPlayer `pause()` / `resume()` (fixed issue where only video paused but audio continued playing)

---

## v0.4.0

**Release Date**: April 2026

### Breaking Changes
- `play()`, `finish()` changed to `suspend fun` (must be called within a coroutine)
- `KlleonOndeviceSdk.stopping` companion property removed, replaced with `isStopping` instance property

### Changes
- Removed `runBlocking` from `BaseElement.stop()`: `withTimeout(2s)` + forced cancel fallback
- Introduced `stopMutex`: prevents concurrent `stop()` entry (ANR/deadlock/double release prevention)
- Added `awaitFinish()`: prevents resource contention during fast re-entry (finish → play)
- Removed unused `bridgeJob` code

---

## v0.3.6

**Release Date**: April 2026

### Changes
- Added `avatarId` parameter to `ResourceManager.init()`
- Separated YUV buffer lifecycle: buffer reuse for same avatar re-entry (0ms)
- 3-second cooldown when switching avatars (OOM protection for low-end TV SoCs)
- Buffer retained on `finish()` (`resetState`) + full release on app exit (`deinit`)
- Automatic background memory release via `onTrimMemory` / `onLowMemory`
- Internal IO coroutine transition in `init()` (removed Main thread blocking)
