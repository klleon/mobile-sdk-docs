### 1. 권한 관리
- SDK 초기화 전에 항상 권한을 확인하세요
- 사용자 친화적인 메시지로 권한 거부를 처리하세요
- 필요한 권한에 대한 명확한 근거를 제공하세요
- 다양한 Android 버전에서 권한 플로우를 테스트하세요

```kotlin
private fun showPermissionRationale() {
    Toast.makeText(
        this,
        "이 앱이 제대로 작동하려면 오디오 권한이 필요합니다.",
        Toast.LENGTH_LONG
    ).show()
}
```

### 2. 리소스 관리
- `ResourceManager.download()`를 사용하여 에셋을 캐시 디렉토리로 복사하세요
- `download()` 완료 후 `play()`를 호출하세요
- SDK 사용 완료 시 `finish()`를 호출하여 리소스를 정리하세요
- `onDestroy()`에서 리소스를 정리하세요

```kotlin
override fun onDestroy() {
    super.onDestroy()
    klleonOndeviceSdk.finish()
}
```

### 3. 상태 관찰
- ViewModel을 사용하여 SDK 상태 변경을 관찰하세요
- 코루틴 스코프에서 상태 업데이트를 수집하세요
- try-catch 블록으로 JSON 응답을 안전하게 파싱하세요
- UI 변경을 위해 메인 스레드에서 상태 업데이트를 처리하세요

### 4. 오류 처리
- SDK 메서드 호출을 try-catch 블록으로 감싸세요
- 디버깅을 위해 오류를 적절히 로깅하세요
- 실패한 작업에 대해 사용자 피드백을 제공하세요
- SDK 메서드 호출 전에 입력을 검증하세요

```kotlin
fun safeOperation() {
    try {
        klleonOndeviceSdk.sendMessage(message)
    } catch (e: Exception) {
        Log.e("SDK", "작업 실패", e)
        showUserError("작업이 실패했습니다. 다시 시도해 주세요.")
    }
}
```

### 5. UI 통합
- Compose에서 `AndroidView`를 사용하여 `KlleonSdkView`를 통합하세요
- SDK 메서드 호출 전에 뷰가 제대로 초기화되었는지 확인하세요
- 뷰 라이프사이클을 적절히 처리하세요
- 다양한 화면 크기와 방향에서 UI를 테스트하세요

### 6. 음성 인식 통합

버튼 탭 시 음성 인식을 시작하고, 무음 감지 시 자동 종료됩니다.

```kotlin
// 음성 인식 시작
val success = klleonOndeviceSdk.startVoiceRecognition()

// 결과는 sdkState Flow로 수신
klleonOndeviceSdk.sdkState.collect { state ->
    val json = JSONObject(state)
    when (json.optString("chat_type")) {
        "STT_RESULT" -> {
            // 음성 인식 최종 결과
            val text = json.getString("message")
        }
        "STT_PARTIAL" -> {
            // 중간 결과 (실시간 표시용)
            val partialText = json.getString("message")
        }
    }
}
```

### 7. 발화 중단

아바타가 발화 중일 때 즉시 중단할 수 있습니다.

```kotlin
klleonOndeviceSdk.sendStopSpeech()
```

### 8. Android TV 대응

TV 디바이스에서는 다음 사항을 고려하세요:

- **화면 회전**: `KlleonSdkView.setRenderRotation(270)` 으로 270도 회전 지원
- **리모컨 키 매핑**: 회전된 화면에 맞게 방향키를 매핑
- **포커스 관리**: TV 자동 포커스로 인한 키보드 자동 표시 방지

```kotlin
// TV용 화면 회전
klleonSdkView.setRenderRotation(270)
```

## 문제 해결

### 일반적인 문제

#### 1. 권한 거부됨
**문제**: `RECORD_AUDIO` 권한이 허용되지 않음

**해결책**:
- `AndroidManifest.xml`에 권한이 선언되어 있는지 확인
- 권한 처리 로직 확인
- 다양한 Android 버전에서 테스트

#### 2. SDK 상태가 업데이트되지 않음
**문제**: 상태 관찰이 작동하지 않음

**해결책**:
- ViewModel에서 상태 관찰이 제대로 설정되어 있는지 확인
- 코루틴 스코프가 활성화되어 있는지 확인
- 상태 수집이 시작되었는지 확인

#### 3. 뷰가 준비되지 않음
**문제**: `play()` 호출 전에 `KlleonSdkView`가 초기화되지 않음

**해결책**:
- `AndroidView` 팩토리에서 뷰 초기화 확인
- 뷰가 준비되었을 때만 버튼 활성화
- SDK 메서드 호출 전에 null 검사 추가

#### 4. USB 마이크가 인식되지 않음
**문제**: USB 마이크를 연결했으나 SDK에서 사용하지 않음

**해결책**:
- `hasUsbMicrophone()`으로 연결 상태 확인
- `mmapEnabled = false` (기본값) 확인 — MMAP 활성화 시 일부 USB 마이크 비호환
- 디바이스 로그 확인: `adb logcat | grep "Ondevice_Mic"`

#### 5. 아바타 발화 중 음성 인식 안 됨
**문제**: 아바타가 말하는 동안 음성 인식이 시작되지 않음

**해결책**:
- 에코 방지를 위해 아바타 발화 중에는 음성 인식이 차단됩니다
- `sendStopSpeech()`로 아바타 발화를 먼저 중단하거나, 발화 완료 후 시도하세요

### 디버깅 팁

#### 상태 모니터링
```kotlin
// 모든 상태 변경 모니터링
klleonOndeviceSdk.sdkState.collect { state ->
    Log.d("SDKState", "상태 업데이트: $state")

    try {
        val json = JSONObject(state)
        val message = json.getString("message")
        val chatType = json.optString("chat_type", "unknown")
        Log.d("SDKState", "파싱됨 - chatType: $chatType, 메시지: $message")
    } catch (e: Exception) {
        Log.w("SDKState", "상태 JSON 파싱 실패", e)
    }
}
```

#### 녹음기 상태 모니터링
```kotlin
// 녹음기 상태 변경 모니터링
klleonOndeviceSdk.recorderState.collect { state ->
    when (state) {
        is OboeRecorderState.Recording -> {
            Log.d("Recorder", "녹음 중 - 장치: ${state.deviceName}")
        }
        is OboeRecorderState.Error -> {
            Log.e("Recorder", "에러: ${state.message}")
        }
        else -> Log.d("Recorder", "상태: $state")
    }
}
```
