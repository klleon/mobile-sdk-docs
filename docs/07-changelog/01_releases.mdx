# 릴리스 노트

Klleon Ondevice SDK의 버전별 변경사항입니다.

## v0.6.0 (Latest)

**릴리스일**: 2026년 5월

### 새로운 기능
- **Dual YUV face crops 런타임 적용** (LIPSYNC-007): `ResourceManager.init()`에 `faceCropYuvFileName`, `faceCropYuvMetaFileName` 파라미터 추가 (둘 다 default `""` — backward-compat). 사전 추출된 face crop YUV가 있으면 `warpAffine` 스킵하여 추론 경로 단축.
- `ResourceManager` public properties 추가: `faceCropYuvBuffer`, `faceCropYuvPath`, `faceCropYuvMetaPath`

### 변경사항
- `VideoDecoder.decodeVideo` 콜백 시그니처 확장: `(rgba, faceFrame, frame, total)` (이전 `(rgba, frame, total)`) — 내부 변경, 공개 API 영향 없음
- 립싱크 캘리브레이션 도구 (`tools/lipsync_quality/`) 정비

## v0.5.0

**릴리스일**: 2026년 4월

### Breaking Changes
- `pauseVideo()` / `resumeVideo()` / `isVideoPaused()` → `enterStandby()` / `exitStandby()` / `isStandby()`로 리네이밍
- `registerKeywords()` / `clearKeywords()` / `keywordDetected` → `registerWakewords()` / `clearWakewords()` / `wakewordDetected`로 리네이밍

### 새로운 기능
- 대기화면(Standby) 모드: 파이프라인 전체 flush + 타이밍 재초기화로 빨리감기/ANR 방지
- Wake-word 매칭: STT 인식 결과에서 등록된 키워드 감지 → 콜백 전달 (standby 중에만 활성)
- `setIdleTimeout()` / `disableIdleTimeout()`: TTS_END 후 무활동 시 자동 대기화면 진입
- `isResponding: StateFlow<Boolean>`: 아바타 응답 상태 추적 (PREPARING_RESPONSE ~ TTS_END)
- `STANDBY_ENTER` 이벤트: 자동 대기화면 진입 시 sdkState로 발행

### 버그 수정
- VideoDecoder `stop()` race condition (pauseToken/stopToken 순서 역전)
- ASYNC 모드 consumer가 pauseToken 미확인
- AudioPlayer `pause()` / `resume()` 추가 (비디오만 멈추고 오디오 계속 재생 문제)

## v0.4.0

**릴리스일**: 2026년 4월

### Breaking Changes
- `play()`, `finish()` → `suspend fun`으로 변경 (코루틴 내에서 호출 필요)
- `KlleonOndeviceSdk.stopping` companion 프로퍼티 제거 → `isStopping` 인스턴스 프로퍼티로 대체

### 변경사항
- `BaseElement.stop()` runBlocking 제거: `withTimeout(2s)` + 강제 cancel 폴백
- `stopMutex` 도입: `stop()` 동시 진입 방어 (ANR/데드락/double release 방지)
- `awaitFinish()` 추가: 빠른 재진입(finish → play) 시 리소스 경합 방지
- 미사용 `bridgeJob` 코드 제거

## v0.3.6

**릴리스일**: 2026년 4월

### 변경사항
- `ResourceManager.init()`에 `avatarId` 파라미터 추가
- YUV buffer lifecycle 분리: 동일 아바타 재진입 시 버퍼 재사용 (0ms)
- 다른 아바타 전환 시 3초 쿨다운 (저사양 TV SoC OOM 방어)
- `finish()` 시 버퍼 유지 (`resetState`) + 앱 종료 시 완전 해제 (`deinit`)
- `onTrimMemory` / `onLowMemory` 기반 백그라운드 메모리 자동 해제
- `init()` 내부 IO 코루틴 전환 (Main 스레드 블로킹 제거)
