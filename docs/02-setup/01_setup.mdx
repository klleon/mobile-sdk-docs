# 설정

## 1) 요구사항
- **최소 SDK**: 26 (Android 8.0)
- **타겟 SDK**: 36
- **지원 ABI**: arm64-v8a, armeabi-v7a

### 스펙 요구사항

| 항목 | 최소 스펙 | 비고 |
|------|----------|------|
| Android | API 26 (8.0) 이상 | |
| CPU | ARM 64bit (ARMv8) | 4코어 이상 |
| RAM | 4GB 이상 | |
| GPU | OpenCL 1.2 + `cl_khr_fp16` | GPU 추론 필수 |
| GPU 성능 | FP32 ~200 GFLOPS 이상 | 기준: Galaxy S9 (Mali-G72 MP18, ~247 GFLOPS) |

:::warning 사전검증 필요
카탈로그 스펙만으로는 SDK 동작을 보장할 수 없습니다.

- **같은 수치, 다른 결과**: 비슷한 성능(GFLOPS)의 GPU여도 벤더(Mali/Adreno/PowerVR)에 따라 추론 시간이 2~3배 차이나는 사례가 확인되었습니다.
- **GPU보다 CPU가 더 중요**: SDK 전체 처리의 약 2/3가 CPU 작업입니다. GPU 스펙만 보고 판단하면 실제 성능을 예측할 수 없습니다.
- **드라이버 호환성**: OpenCL 드라이버 버그로 GPU delegate 초기화가 실패하는 기기가 존재합니다. 같은 모델이라도 OS 버전에 따라 동작이 달라질 수 있습니다.
- **발열 쓰로틀링**: 장시간 동작 시 발열로 인해 CPU/GPU 클럭이 낮아져 성능이 저하됩니다.
- **배포 후 문제 발견 시 비용이 큼**: 키오스크/TV 등 대량 배포 환경에서 문제 발생 시 현장 대응 비용이 발생합니다. 5분 벤치마크로 사전에 확인하면 이 리스크를 제거할 수 있습니다.

**신규 디바이스 도입 전 벤치마크 앱으로 사전검증을 권장합니다.**

| 항목 | 합격 기준 |
|------|----------|
| OpenCL 지원 | OpenCL 1.2+ & `cl_khr_fp16` |
| TFLite GPU 추론 | 초기화 성공 |
| 추론 시간 | &lt;60ms (전처리/후처리 제외) |
| FPS (발화 시) | 24fps 이상 유지 |
| 5분 연속 동작 | 메모리 안정, 발열 &lt;45°C |
:::

### 리소스

SDK에는 비디오 파일(`.mp4`)과 얼굴 모델 파일(`.npz`)이 필요합니다.
이 파일들은 `assets`(`src/main/assets`) 폴더에 배치한 뒤, `ResourceManager.init()`을 호출하여 캐시 복사 및 초기화합니다.

```kotlin
import io.klleon.ondevice.ResourceManager

val resourceManager = ResourceManager()

ResourceManager.init(
    context = this,
    mp4FileName = "loop_sample.mp4",
    npzFileName = "loop_sample.npz",
    avatarId = "character_001",
    onSuccess = {
        // 리소스 준비 완료 — play() 호출 가능
    },
    onError = { error ->
        Log.e("Setup", "리소스 준비 실패", error)
    }
)
```

:::info
`init()` 완료 후 내부 경로가 자동 설정됩니다.
`play()` 호출 전에 반드시 `init()`의 `onSuccess` 콜백이 호출되어야 합니다.
:::

## 2) Android 구성

```kotlin
android {
    compileSdk = 36

    defaultConfig {
        minSdk = 26
        targetSdk = 36
    }

    packaging {
        jniLibs {
            pickFirsts += "**/libc++_shared.so"
        }
    }
}
```

## 3) Gradle 의존성

앱 레벨 `build.gradle.kts`에 다음 의존성을 추가합니다:

```kotlin
dependencies {
    // SDK
    implementation("io.klleon:ondevice-sdk:0.5.0")

    // Coroutines (SDK 필수)
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-core:1.9.0")
    implementation("org.jetbrains.kotlinx:kotlinx-coroutines-android:1.9.0")
}
```

:::info 번들 라이브러리
SDK AAR에 다음 라이브러리들이 포함되어 있습니다:
- OkHttp, Okio, Retrofit
- Jackson (Core, Module)
- TensorFlow Lite
- OpenCV
- PyTorch Mobile
- Oboe
- ONNX Runtime
- Agora RTC

앱에서 OkHttp 또는 Okio를 사용하는 라이브러리(예: Coil)가 있으면 버전 충돌이 발생할 수 있습니다. 이 경우 `RuntimeClasspath`에서 해당 그룹을 exclude하세요:
```kotlin
configurations.matching { it.name.contains("RuntimeClasspath", ignoreCase = true) }.configureEach {
    exclude(group = "com.squareup.okhttp3")
    exclude(group = "com.squareup.okio")
}
```
:::

## 4) ProGuard / R8

SDK AAR에 필요한 ProGuard/R8 규칙이 포함되어 있어 **별도의 설정이 필요 없습니다.**

`UnsatisfiedLinkError` 발생 시 위 Android 구성의 `jniLibs.pickFirsts` 설정을 확인하세요.
